## Core Methodology

### Step 1: Document Analysis
When user provides a document (Markdown/TXT/DOCX/PDF):
1. Extract core value proposition
2. Identify target audience
3. Map key features/benefits
4. Define conversion goals

### Step 3: Information Architecture
Build page structure based on Julian Shapiro's conversion layers:

```
Hero Section (2-3 variants for A/B test)
├── Above-fold hook (Bold Claim / Objection Killer / Story)
├── Social proof anchor
└── Primary CTA

Value Stack
├── Problem agitation
├── Solution presentation
├── Feature/benefit mapping
└── Differentiation block

Trust Layer
├── Social proof (logos, testimonials)
├── Risk reversal (guarantees)
└── Objection handlers

Conversion Layer
├── Pricing/value clarity
├── FAQ (objection prevention)
└── Final CTA

Technical Foundation
├── SEO meta structure
├── Schema markup
└── Performance optimization
```

### Step 4: Growth Integration

**A/B Test Framework** - Generate 3 Hero variants:

1. **Bold Claim**: "The [product] that [competitors] don't want you to know about"
   - Best for: Disruptive products, contrarian positioning
   
2. **Objection Killer**: "Finally, a [product] that [solves specific pain point] without [common objection]"
   - Best for: Skeptical audiences, overcoming switching costs
   
3. **Story**: "When [persona] started using [product], [specific outcome happened in time frame]"
   - Best for: Emotional connection, complex products

**Growth Loops** - Design one of:
- **Content Loop**: User-generated content → SEO traffic → New users → More content
- **Product Loop**: Usage → Value created → Sharing → New users → More usage
- **Community Loop**: Engagement → Network effects → Retention → Advocacy → New users

**Content-Led SEO**:
- Topic clusters over keyword stuffing
- Pain-point driven content architecture
- Structured data implementation
- Internal linking strategy

Use primitives-first approach:

```css
/* Primitives */
:root {
  /* Color */
  --color-primary: [archetype-based];
  --color-secondary: [trend-based];
  --color-accent: [conversion-optimized];

  --font-display: [archetype-appropriate];
  --font-body: [readability-first];

  /* Layout */

```

### Step 6: Quality Assurance

**SEO Checklist**:
- [ ] Title tag ≤ 60 chars with primary keyword
- [ ] Meta description ≤ 160 chars with CTA
- [ ] H1 contains primary keyword
- [ ] Schema markup for Product/Article/Organization
- [ ] Open Graph tags complete
- [ ] Canonical URL set
- [ ] Image alt tags descriptive
- [ ] Internal links (3-5 per 1000 words)
- [ ] Page speed < 2s
- [ ] Mobile responsive
- [ ] Core Web Vitals pass

**Conversion Checklist**:
- [ ] Hero CTA above fold
- [ ] 3 Hero variants for testing
- [ ] Social proof within first 2 screens
- [ ] Risk reversal visible
- [ ] Pricing clarity (if applicable)
- [ ] Single primary CTA per section
- [ ] Friction points minimized
- [ ] Form fields ≤ 4 (if applicable)

## Anti-Patterns (Avoid These)

❌ **Aesthetic Anti-Patterns**:
- Trends without purpose
- Inconsistent spacing
- Poor typography hierarchy
- Accessibility neglect

❌ **Conversion Anti-Patterns**:
- Multiple competing CTAs
- Hidden pricing
- Long forms without progress indication
- Generic social proof
- Vague value propositions

❌ **Growth Anti-Patterns**:
- Funnel-only thinking
- No retention mechanism
- Missing viral loops
- Keyword stuffing
- Duplicate content
